1. File-System Management

What is a File System?

The file system provides the mechanism for organizing, storing, retrieving, and managing data on storage devices. It is one of the most visible components of an operating system.

The OS provides a uniform, logical view of information storage, abstracting the physical properties of storage devices into a logical unit: the file.

Files

A file is a named collection of related information stored on secondary storage:

  • Text files - Sequences of characters organized into lines
  • Binary files - Programs, images, multimedia content
  • Attributes - Name, type, size, permissions, timestamps
  • Operations - Create, read, write, delete, seek, open, close

Directories

Directories organize files into a hierarchical structure:

  • Root directory - The top-level directory in the hierarchy
  • Subdirectories - Directories within directories
  • Path - The route from root to a specific file (e.g., /home/user/document.txt)

Directory Tree Structure

A typical Unix/Linux directory tree with root at the top and subdirectories branching downward.

OS File-System Activities

The operating system is responsible for the following file-system management activities:

  • Creating and deleting files and directories
  • Mapping files onto secondary storage
  • Supporting primitives for manipulating files and directories
  • Backup of files onto stable (non-volatile) storage media

Key Insight: The file system hides the complexity of physical storage devices, providing users and programs with a simple, uniform interface to store and retrieve data.

2. Mass-Storage Management

Why Mass-Storage Management?

Since main memory is too small and volatile to hold all data permanently, the OS must manage secondary storage (disks) efficiently. Most programs are stored on disk until loaded into memory.

OS Mass-Storage Activities

Disk Management

  • Mounting and unmounting storage devices
  • Free-space management - Tracking available disk space
  • Storage allocation - Assigning disk blocks to files
  • Disk scheduling - Ordering disk requests for efficiency
  • Partitioning - Dividing disks into logical sections

Storage Allocation Methods Comparison

Method Approach Advantages Disadvantages
Contiguous Files stored in consecutive blocks Fast sequential & random access External fragmentation, file size must be known
Linked Each block points to the next No external fragmentation, easy growth Slow random access, pointer overhead
Indexed Index block holds all pointers Supports direct access, no fragmentation Index block overhead, limited file size

Tertiary Storage

Beyond primary and secondary storage, tertiary storage provides archival solutions:

  • Optical discs (CD, DVD, Blu-ray) - Read-only or read-write
  • Magnetic tapes - High capacity, sequential access, used for backups
  • Cloud storage - Remote storage accessed over the network

Performance Impact: The speed of the entire system often depends on how efficiently the OS manages disk storage, since disk I/O is one of the major bottlenecks in computing.

3. Caching

What is Caching?

Caching is the technique of copying frequently accessed data into a faster storage medium. It is a fundamental principle used at every level of a computer system to improve performance.

When data is needed, it is first checked in the cache. If found (cache hit), data is used directly from cache. If not found (cache miss), data is fetched from the slower source and a copy is placed in the cache for future use.

Multi-Level Caching Flow

L1 Cache ~1 ns | 32-64 KB
→
L2 Cache ~4 ns | 256 KB-1 MB
→
L3 Cache ~10 ns | 4-64 MB
↓
Main Memory (RAM) ~100 ns | 8-64 GB
→
Disk (SSD/HDD) ~10 μs - 10 ms | TB

Data flows through multiple cache levels. Each miss at a faster level triggers a fetch from the next slower level.

Key Caching Concepts

Cache Management

  • Cache size - Limited; careful selection of what to cache is critical
  • Replacement policy - Decides which cached data to evict (e.g., LRU, FIFO)
  • Write policy - Write-through (immediate) vs write-back (deferred)
  • Coherency - Ensuring all copies of data are consistent across caches

Caching in Everyday Computing

  • CPU cache - Hardware caches instructions and data near the processor
  • Disk cache - OS caches frequently accessed disk blocks in RAM
  • Web cache - Browsers cache web pages and images locally
  • DNS cache - OS caches domain name resolutions

Important: Caching introduces the cache coherency problem in multiprocessor systems. When multiple CPUs cache the same data, updates by one CPU must be reflected in all other caches to maintain consistency.

4. I/O Subsystem

What is the I/O Subsystem?

The I/O subsystem manages and controls all input/output operations and devices. One of its key purposes is to hide the peculiarities of specific hardware devices from the rest of the operating system.

I/O System Layer Stack

User Application Issues I/O requests via system calls
Kernel I/O Subsystem Scheduling, buffering, caching, spooling, error handling
Device Drivers Device-specific code that interfaces with hardware
Device Controllers Hardware interface (registers, buffers)
Physical Devices Keyboard, disk, network, display, etc.

I/O Kernel Services

I/O Techniques Comparison

Technique Description Example
Buffering Temporarily storing data in memory during transfer between devices with speed mismatches Streaming video buffers frames ahead of playback
Caching Keeping a copy of frequently used data in faster memory Disk block cache in RAM
Spooling Holding output for a device that cannot accept interleaved data streams Print spooler queues documents for the printer

Device Drivers

Device drivers provide a uniform interface to the I/O subsystem:

  • Each device type has its own driver
  • Drivers encapsulate device-specific details
  • Allow the OS to treat all devices through a common interface
  • Can be loaded dynamically (plug-and-play)

Design Principle: The I/O subsystem provides a device-independent interface. Programs can read/write to any device using the same system calls, regardless of whether it's a disk, network card, or keyboard.

5. Protection and Security

Protection vs Security

Protection is an internal mechanism: controlling access of processes and users to system resources. Security is defense against external and internal attacks: viruses, worms, denial-of-service, identity theft, etc.

Security Layers

Physical Security Data centers, locks, access cards
Network Security Firewalls, encryption, VPN
OS Security Authentication, access control
Application Input validation

Security is implemented in layers. A breach must penetrate multiple layers to reach the core application data.

Protection Mechanisms

User Authentication

The OS must distinguish among users to determine who can do what:

  • User ID (UID) - Unique identifier for each user
  • Group ID (GID) - Users can belong to groups for shared access
  • Passwords and credentials - Verify user identity
  • Privilege escalation - Temporarily gaining higher privileges (e.g., sudo)

Access Control

Files and resources have permissions that control who can access them:

  • Read (r) - View file contents
  • Write (w) - Modify file contents
  • Execute (x) - Run as a program

Unix File Permissions Example

The command ls -l shows permissions in the format:

-rwxr-xr-- 1 owner group 4096 Jan 1 file.txt
rwx Owner
r-x Group
r-- Others

Defense in Depth: Security must be implemented at multiple levels - physical, human, operating system, and network. A chain is only as strong as its weakest link.

6. Virtualization

What is Virtualization?

Virtualization is a technology that allows a single physical machine to run multiple operating systems concurrently. A Virtual Machine Manager (VMM) or hypervisor creates and manages virtual machines (VMs), each with its own virtual hardware.

VMM Architecture

Types of Hypervisors

Hypervisor Comparison

Feature Type 1 (Bare-Metal) Type 2 (Hosted)
Runs on Directly on hardware On top of a host OS
Performance Better (less overhead) Slightly lower (extra OS layer)
Use case Servers, data centers Desktop testing, development
Examples VMware ESXi, Hyper-V, Xen VirtualBox, VMware Workstation

Benefits of Virtualization

  • Server consolidation - Run multiple servers on one physical machine
  • Isolation - VMs are isolated from each other; a crash in one doesn't affect others
  • Testing and development - Run different OS environments simultaneously
  • Snapshots and migration - Save VM state and move VMs between physical hosts
  • Cloud computing - Foundation of modern cloud infrastructure

Key Insight: Virtualization is the foundation of cloud computing. Services like AWS, Azure, and Google Cloud use hypervisors to run thousands of virtual machines on shared physical infrastructure.

7. Distributed Systems

What are Distributed Systems?

A distributed system is a collection of physically separate, networked computers that work together to appear as a single coherent system. They communicate and coordinate through a network.

Network Types

Network Types Comparison

PAN

Personal Area Network

Range: ~1-10 meters

Bluetooth, USB connections

LAN

Local Area Network

Range: Building / Campus

Ethernet, Wi-Fi

MAN

Metropolitan Area Network

Range: City / Region

Cable TV networks, city Wi-Fi

WAN

Wide Area Network

Range: Countries / Global

The Internet, leased lines

Why Distributed Systems?

  • Resource sharing - Share hardware, software, and data across network
  • Computation speedup - Distribute tasks across multiple machines
  • Reliability - If one node fails, others can continue operating
  • Communication - Enable users to exchange information (email, messaging)

Network Operating Systems vs Distributed Operating Systems

  • Network OS - Users are aware of multiple machines; must explicitly access remote resources (e.g., SSH, file sharing)
  • Distributed OS - Users see a single system; resource access is transparent (the system decides where to execute tasks)

Modern Relevance: Today's internet services (search engines, social media, streaming) are all built on distributed systems, with thousands of computers working together seamlessly.

8. Computer System Architecture

Computer systems can be categorized by the number and arrangement of their processors:

Architecture Comparison

Single Processor

Description: One general-purpose CPU with one processing core

Characteristics:

  • Simplest architecture
  • May have special-purpose processors (GPU, DMA controller)
  • Historically dominant, now rare
SMP (Symmetric Multiprocessing)

Description: Multiple processors of equal capability sharing memory and I/O

Characteristics:

  • Each CPU runs any process
  • Shared memory bus
  • Most common modern architecture
NUMA (Non-Uniform Memory Access)

Description: Multiple CPUs, each with local memory, connected via shared bus

Characteristics:

  • Fast access to local memory
  • Slower access to remote memory
  • Scales better than SMP
Clustered Systems

Description: Multiple independent systems (nodes) working together

Characteristics:

  • High availability and fault tolerance
  • Can be asymmetric or symmetric clustering
  • Used in servers and supercomputers

Multicore Processors

Modern processors contain multiple cores on a single chip:

  • Each core is a full processing unit
  • Cores share on-chip cache (typically L2/L3)
  • More energy-efficient than multiple separate processors
  • On-chip communication is faster than between-chip

Real-World Architecture Examples

  • Your laptop/phone - Multicore SMP (4-16 cores)
  • Server - Multi-socket NUMA (2-8 sockets, each with 32+ cores)
  • Google/Amazon data center - Clustered system with thousands of nodes
  • Supercomputer - Massively clustered with millions of cores

Trend: The trend in computing is toward more cores and more parallelism. Operating systems must efficiently manage and schedule work across many processors and cores.

Summary and Key Takeaways

What We've Learned

  1. File-system management provides organization - The OS manages files and directories, providing a logical structure on top of physical storage
  2. Mass-storage management is critical - Efficient disk management (allocation, scheduling) directly impacts system performance
  3. Caching improves performance at every level - From CPU caches to web caches, storing frequently used data in faster storage is a universal optimization
  4. The I/O subsystem hides hardware complexity - Layered architecture with device drivers provides a uniform interface for all devices
  5. Protection controls internal access - User IDs, permissions, and access control lists determine who can access what resources
  6. Security defends against attacks - Multiple layers of defense protect the system from external and internal threats
  7. Virtualization enables resource sharing - Hypervisors allow multiple OS instances on one physical machine, forming the basis of cloud computing
  8. System architectures vary in scale - From single processors to clustered systems, each architecture has trade-offs in performance, cost, and complexity
