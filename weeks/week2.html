1. Storage Structure

Computer systems need different types of storage to operate efficiently. The two main categories are:

Main Memory (RAM)

Random Access Memory is the primary storage that the CPU can access directly:

  • Volatile - Contents are lost when power is turned off
  • Fast Access - Can be read/written quickly by CPU
  • Limited Size - Typically 8GB to 64GB in modern computers
  • Directly Addressable - CPU uses memory addresses to access data

Secondary Storage

Provides non-volatile storage for programs and data:

  • Hard Disk Drives (HDD) - Magnetic storage, high capacity, slower
  • Solid State Drives (SSD) - Flash memory, faster than HDD
  • Non-volatile - Data persists even when power is off
  • Large Capacity - Can store terabytes of data

Key Insight: Programs must be loaded from secondary storage into main memory before they can be executed. The CPU cannot directly execute programs stored on disk.

2. Storage Definitions

Understanding storage requires knowing the basic units of measurement:

Basic Units

A bit is the smallest unit of data (0 or 1). A byte is a group of 8 bits.

Storage Units Table

Unit Symbol Size (Bytes) Approximate Value
Kilobyte KB 210 1,024 bytes (~1 thousand)
Megabyte MB 220 1,048,576 bytes (~1 million)
Gigabyte GB 230 ~1 billion bytes
Terabyte TB 240 ~1 trillion bytes
Petabyte PB 250 ~1 quadrillion bytes

Real-World Examples

  • 1 KB - A short email or text document
  • 1 MB - A high-quality photo or a minute of MP3 audio
  • 1 GB - About 250 MP3 songs or a short HD video
  • 1 TB - About 500 hours of HD video

3. Storage Hierarchy

Storage devices are organized in a hierarchy based on speed, cost, and capacity:

Storage Hierarchy Pyramid

Registers Fastest, Most Expensive, Smallest
Cache (L1, L2, L3) Very Fast, Expensive, Small (KB-MB)
Main Memory (RAM) Fast, Moderate Cost, Medium (GB)
Solid State Drive (SSD) Moderate Speed, Lower Cost, Large (TB)
Hard Disk Drive (HDD) Slower, Cheapest, Largest (TB)

As we go down the pyramid: Speed decreases, Capacity increases, Cost per byte decreases

Key Concepts

Caching

Caching is the process of copying frequently accessed data to faster storage:

  • Data is copied from slower storage to faster storage
  • When data is needed, faster storage is checked first
  • Improves overall system performance significantly
  • Used at multiple levels (CPU cache, disk cache, web cache)

Volatility

  • Volatile Storage - Loses data when power is off (Registers, Cache, RAM)
  • Non-volatile Storage - Retains data without power (SSD, HDD)

Trade-off: There is always a trade-off between speed, cost, and capacity. Faster storage is more expensive and has less capacity. The hierarchy allows systems to balance these factors.

4. Direct Memory Access (DMA)

What is DMA?

Direct Memory Access is a feature that allows I/O devices to transfer data directly to/from memory without involving the CPU for each byte.

DMA Data Transfer

CPU
Memory
DMA Controller
I/O Device
1. CPU initiates transfer
2. DMA controller handles data transfer
3. CPU is free to do other work
4. DMA interrupts CPU when done

Benefits of DMA

  • Frees the CPU - CPU can perform other tasks during data transfer
  • High-speed I/O - Data moves at memory speed, not CPU instruction speed
  • One interrupt per block - Instead of one interrupt per byte
  • Efficient for large transfers - Essential for disk and network operations

Without DMA vs With DMA

Without DMA: CPU must read each byte from device, store it in memory. For 1MB transfer at 1 byte per instruction = 1 million CPU cycles wasted.

With DMA: CPU sets up transfer once, DMA controller handles the rest. CPU uses only a few cycles for setup and completion interrupt.

5. Operating System Operations

When a computer starts, several key operations occur:

Bootstrap Program

The bootstrap program is the first code that runs when the computer powers on:

  • Stored in ROM or EEPROM (firmware)
  • Initializes CPU registers, device controllers, and memory
  • Locates and loads the OS kernel into memory
  • Starts execution of the kernel

System Daemons

Daemons are background processes that start during boot and run continuously:

  • Provide system services (printing, networking, scheduling)
  • Run outside the kernel but support system operations
  • Examples: sshd (secure shell), httpd (web server), cron (scheduler)

Interrupt-Driven Operation

Modern operating systems are interrupt driven:

  • Hardware interrupts - Generated by devices (keyboard, disk, network)
  • Software interrupts (Traps) - Generated by programs (system calls, errors)
  • Interrupts signal the OS that an event needs attention
  • Allows efficient handling of I/O and other events

6. Multiprogramming

What is Multiprogramming?

Multiprogramming keeps multiple programs in memory simultaneously to increase CPU utilization. When one program waits for I/O, another can use the CPU.

How It Works

  1. Multiple jobs (programs) are loaded into memory
  2. OS selects one job to start executing (job scheduling)
  3. When the running job needs to wait (e.g., for I/O), OS switches to another job
  4. This keeps the CPU busy, improving utilization

Goal: Maximize CPU utilization by ensuring there is always a job ready to run. This is especially important for batch systems where jobs are submitted and run without user interaction.

7. Multitasking (Timesharing)

What is Multitasking?

Multitasking (or timesharing) extends multiprogramming by rapidly switching the CPU between processes, giving each user the illusion of having their own dedicated computer.

Key Concepts

Process

A process is a program in execution. It includes:

  • The program code
  • Current activity (program counter, registers)
  • Process stack (temporary data)
  • Data section (global variables)
  • Heap (dynamically allocated memory)

CPU Scheduling

When multiple processes are ready to run, the OS must choose which one gets the CPU:

  • Time quantum - Each process gets a small time slice
  • Context switch - Saving and restoring process state
  • Scheduling algorithms - Rules for selecting the next process

Response Time

Multitasking aims to provide reasonable response time:

  • Users expect quick responses to their input
  • Time slices are typically 10-100 milliseconds
  • Switching happens so fast users don't notice

Real-World Example

When you have a web browser, music player, and word processor open simultaneously, multitasking allows all three to appear to run at the same time. The CPU rapidly switches between them, giving each a turn.

Memory Management Requirements

  • Allocation - Assigning memory to processes
  • Protection - Preventing processes from accessing each other's memory
  • Relocation - Allowing programs to run at different addresses
  • Sharing - Allowing processes to share code (e.g., libraries)

9. Dual-mode Operation

Why Dual-mode?

Operating systems use dual-mode operation to protect the system from errant or malicious user programs. The hardware distinguishes between user mode and kernel mode.

User Mode vs Kernel Mode

User Mode

Mode bit = 1

  • Limited access to hardware
  • Cannot execute privileged instructions
  • Memory access is restricted
  • Where user applications run

Kernel Mode

Mode bit = 0

  • Full access to hardware
  • Can execute all instructions
  • Unrestricted memory access
  • Where the OS kernel runs

Privileged Instructions

Some instructions can only be executed in kernel mode:

  • I/O instructions (reading/writing to devices)
  • Memory management instructions
  • Interrupt management instructions
  • Instructions that modify the mode bit

Protection Mechanism: If a user program tries to execute a privileged instruction while in user mode, the hardware generates a trap (exception), and the OS can terminate the offending program.

10. Mode Transition

The system transitions between user mode and kernel mode through specific mechanisms:

Mode Transition Flow

User Mode Application executing
System Call / Trap / Interrupt →
Kernel Mode OS handling request
← Return from system call
Launch Interactive Mode Transition Demo

Transition Mechanisms

System Calls

User programs request OS services via system calls:

  • Program executes a special trap instruction
  • Hardware switches to kernel mode
  • OS performs the requested service
  • Control returns to user program in user mode

Interrupts and Traps

  • Interrupt - External event (hardware signal)
  • Trap - Internal event (software-generated, e.g., system call or error)
  • Both cause transition from user mode to kernel mode

Example: Opening a File

  1. User program calls open("file.txt") - User Mode
  2. System call trap occurs → switch to Kernel Mode
  3. OS validates request, accesses file system
  4. OS returns file handle → switch back to User Mode
  5. Program continues with the file handle

11. Process Management

What is a Process?

A process is a program in execution. It is the unit of work in a modern computer system.

Process vs Program

  • Program - Passive entity stored on disk (executable file)
  • Process - Active entity with a program counter and resources
  • One program can create multiple processes (e.g., multiple browser tabs)

Process Resources

A process needs various resources to accomplish its task:

  • CPU time - To execute instructions
  • Memory - For code, data, and stack
  • Files - For input/output operations
  • I/O devices - For communication with peripherals

OS Process Management Activities

The operating system is responsible for:

  • Creating and deleting processes (user and system)
  • Scheduling processes on the CPU
  • Suspending and resuming processes
  • Providing mechanisms for process synchronization
  • Providing mechanisms for process communication
  • Handling deadlocks between processes

Single-threaded vs Multi-threaded: A single-threaded process has one program counter. A multi-threaded process has multiple program counters, allowing multiple locations in the code to be executed simultaneously.

12. Memory Management

Memory management is a critical function of the operating system:

Why Memory Management?

  • All instructions and data must be in memory for execution
  • Multiple processes compete for limited memory
  • CPU can only directly access main memory and registers
  • Efficient memory use improves system performance

Memory Management Activities

The operating system is responsible for:

Tracking Memory Usage

  • Keeping track of which parts of memory are in use
  • Knowing which processes own which memory regions

Allocation and Deallocation

  • Deciding which processes get memory and how much
  • Allocating memory when processes need it
  • Freeing memory when processes terminate or release it

Moving Data

  • Moving data between main memory and disk
  • Implementing virtual memory (using disk as extended memory)
  • Swapping processes in and out of memory

Virtual Memory: Allows programs to use more memory than physically available by using disk space as an extension of RAM. This enables running large programs and more processes simultaneously.

Summary and Key Takeaways

What We've Learned

  1. Storage is hierarchical - From fast, expensive registers to slow, cheap disk drives, with caching to optimize access
  2. DMA enables efficient I/O - Allows data transfer without constant CPU involvement, freeing the CPU for other work
  3. OS operations are interrupt-driven - Hardware and software interrupts allow efficient event handling
  4. Multiprogramming maximizes CPU utilization - By keeping multiple programs in memory and switching when one waits for I/O
  5. Multitasking provides interactivity - Rapid CPU switching gives users the illusion of simultaneous execution
  6. Dual-mode operation provides protection - User mode restricts access, kernel mode has full privileges
  7. Mode transitions occur via system calls and interrupts - Programs request OS services through controlled interfaces
  8. Process management handles program execution - Creating, scheduling, and managing process resources
  9. Memory management is essential - Tracking, allocating, and managing memory for all processes
