1. What is an Operating System?

Definition

An Operating System (OS) is a program that acts as an intermediary between a user of a computer and the computer hardware.

Three Main Goals

Operating systems have three fundamental goals:

  1. Execute user programs - Run applications and solve user problems
  2. Make the computer convenient - Provide an easy-to-use interface
  3. Use hardware efficiently - Manage resources effectively

The OS as an Intermediary

    │    USER     │
    └──────┬──────┘
           │
           ▼
    ┌─────────────┐
    │  OPERATING  │
    │   SYSTEM    │
    └──────┬──────┘
           │

    └─────────────┘

The OS provides a bridge between users and complex hardware

Key Insight: Without an operating system, users would need to directly program hardware using machine code - an extremely difficult and error-prone task. The OS abstracts this complexity.

2. Computer System Structure

A computer system can be divided roughly into four components:

Four Components of a Computer System

1. Users

People, machines, other computers

2. Application Programs

Word processors, compilers, browsers, games

3. Operating System

Controls and coordinates hardware use

4. Hardware

CPU, memory, I/O devices

Hardware

Provides the basic computing resources:

  • CPU (Central Processing Unit) - Executes instructions
  • Memory - Stores data and programs temporarily
  • I/O Devices - Keyboards, displays, disks, network interfaces

Operating System

Controls and coordinates use of hardware among various applications and users. It acts as a resource allocator and control program.

Application Programs

Define the ways in which computing resources are used to solve user problems:

  • Word processors
  • Compilers and development tools
  • Web browsers
  • Database systems
  • Video games

Users

Can be people, other machines, or other computers that interact with the system.

3. Abstract Layered View

The operating system provides an abstraction layer that hides the complexity of hardware from applications and users.

Layered View of Computer System

User
Application Programs
Operating System
Hardware

Each layer only interacts with the layer directly below it

Benefits of Layering

  • Abstraction - Higher layers don't need to know hardware details
  • Modularity - Each layer can be modified independently
  • Security - Lower layers can protect resources from upper layers
  • Portability - Applications can run on different hardware

Example: Writing to a File

When a user saves a document:

  1. User clicks "Save" button
  2. Application calls file write function
  3. OS translates to disk operations
  4. Hardware physically writes data to disk

The user doesn't need to know which disk sector to use!

4. What Operating Systems Do

The role of an operating system varies depending on the point of view:

Three Perspectives on Operating Systems

Personal Computer View

Focus: Ease of use and user experience

Users want their applications to work smoothly. Resource utilization is less important since the computer serves one user.

Shared/Mainframe View

Focus: Resource utilization and fairness

Multiple users share resources. The OS must allocate CPU, memory, and I/O fairly and efficiently.

Mobile/Embedded View

Focus: Battery life and responsiveness

Resources are limited. The OS must optimize for power efficiency while maintaining good user experience.

No Universal Definition: Operating systems exist because they offer a reasonable way to solve the problem of creating a usable computing system. The design varies based on the target environment.

5. OS Definition: Kernel, System Programs, Middleware

Components of an Operating System

Middleware & Applications
System Programs
Kernel

The kernel is always running; system programs and middleware are loaded as needed

Kernel (Core)

The kernel is the one program running at all times on the computer. It is the heart of the operating system.

  • Always loaded in memory
  • Has complete control over the hardware
  • Manages processes, memory, and devices
  • Handles system calls from applications

System Programs

Programs that are associated with the operating system but are not part of the kernel:

  • File management utilities
  • Status information tools
  • Programming language support
  • Program loading and execution
  • Communications programs

Middleware

Software frameworks that provide additional services to application developers:

  • Databases
  • Multimedia services
  • Graphics frameworks

6. Computer System Organization

Modern computers use a common bus architecture to connect all components:

System Bus Architecture

CPU
Memory
Disk Controller
USB Controller
Graphics Adapter
Hard Disk
Keyboard/Mouse
Monitor

All components communicate through the shared system bus

Key Concepts

Device Controllers

Each device has a controller that manages that specific device type:

  • Contains a local buffer and registers
  • Handles device-specific operations
  • Communicates with CPU through interrupts

Device Drivers

Software that provides a uniform interface to access various hardware devices:

  • Translates OS requests to controller commands
  • Each controller type needs its own driver
  • Loaded as part of the kernel or as modules

Concurrent Operation

The CPU and device controllers can execute concurrently:

  • While CPU runs a program, disk can read data
  • Controllers compete for memory access
  • Memory controller synchronizes access

7. Interrupts and Interrupt Handling

What is an Interrupt?

An interrupt is a signal to the CPU that an event has occurred that needs immediate attention. It temporarily halts the current program execution.

Interrupt Timeline

CPU
Program
ISR
Program
I/O
Transfer

When I/O completes, it interrupts the CPU to process the data

Launch Interactive Interrupt Timeline Demo

How Interrupts Work

  1. Interrupt occurs - Device signals completion or error
  2. CPU saves state - Current program counter and registers are saved
  3. Jump to ISR - CPU looks up address in interrupt vector
  4. Execute ISR - Interrupt Service Routine handles the event
  5. Restore state - CPU restores saved state and continues

Types of Interrupts

Type Source Example
Hardware Interrupt External device Keyboard press, disk completion
Software Interrupt (Trap) Program execution System call, division by zero

Why Interrupts Matter: Without interrupts, the CPU would have to constantly check (poll) each device. Interrupts allow efficient, event-driven handling of I/O.

8. I/O Structure: Synchronous vs Asynchronous

There are two main approaches to handling I/O operations:

Synchronous vs Asynchronous I/O

Synchronous I/O

  1. Program requests I/O
  2. Program WAITS (blocked)
  3. I/O operation completes
  4. Program continues

Pros: Simple to program

Cons: CPU sits idle during I/O

Asynchronous I/O

  1. Program requests I/O
  2. Program CONTINUES working
  3. I/O completes (interrupt)
  4. Program handles result

Pros: CPU stays productive

Cons: More complex programming

Launch Interactive I/O Comparison Demo

Real-World Analogy

Synchronous: You call a restaurant and wait on hold until your order is ready. You can't do anything else.

Asynchronous: You order food via an app and continue with your day. The app notifies you when food arrives.

Device-Status Table

The OS maintains a device-status table that tracks:

  • Type of each device
  • Current state (idle, busy)
  • Pending I/O requests
  • Request queue for each device

9. Computer Startup (Bootstrap Process)

What is Bootstrap?

Bootstrap (or "booting") is the process of loading the operating system when the computer is powered on or restarted.

Boot Sequence

1. Power ON - CPU starts at reset vector [OK]
2. BIOS/UEFI loads from ROM/firmware [OK]
3. POST (Power-On Self Test) runs [OK]
4. Boot device identified (disk, USB, network) [OK]
5. Boot loader loaded from boot sector [OK]
6. Boot loader loads OS kernel into memory [OK]
7. Kernel initializes hardware and drivers [OK]
8. System services and user interface start [OK]
Launch Interactive Boot Sequence Demo

Key Components

Bootstrap Loader

A small program stored in ROM or firmware that:

  • Initializes CPU registers and memory
  • Performs hardware diagnostics
  • Locates and loads the OS kernel

BIOS vs UEFI

  • BIOS (Basic Input/Output System) - Traditional firmware, limited to 16-bit mode
  • UEFI (Unified Extensible Firmware Interface) - Modern replacement, supports 64-bit, faster boot

What if Bootstrap Fails?

If the bootstrap process cannot find a valid operating system:

  • Error message displayed (e.g., "Operating system not found")
  • System halts or enters recovery mode
  • User must fix boot configuration or reinstall OS

Summary and Key Takeaways

What We've Learned

  1. Operating Systems are intermediaries between users and hardware, providing convenience and efficiency
  2. Computer systems have four components: Hardware, OS, Application Programs, and Users
  3. The layered view shows how abstraction hides complexity from higher levels
  4. OS perspective varies: Personal (convenience), Shared (resource allocation), Mobile (efficiency)
  5. The kernel is the core - always running, surrounded by system programs and middleware
  6. Modern computers use bus architecture where CPU, memory, and controllers communicate via shared bus
  7. Interrupts enable efficient I/O by signaling events instead of continuous polling
  8. Synchronous I/O blocks while asynchronous I/O allows concurrent processing
  9. Bootstrap loads the OS through a sequence from BIOS/UEFI to kernel initialization
