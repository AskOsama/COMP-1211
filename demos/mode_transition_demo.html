Understanding Mode Transitions

The CPU uses a mode bit to track whether it's executing user code or kernel code. When a user program needs OS services (like file I/O, network access, or memory allocation), it must make a system call. This triggers a controlled transition to kernel mode where the OS can safely access hardware on behalf of the program. The transition mechanism ensures that user programs cannot bypass security checks or directly access protected resources.
